# 디자인 가이드

프로젝트 전반의 일관성 있는 UI/UX를 유지하기 위한 퍼블리싱 및 디자인 표준 규칙을 정의합니다.
AI 에이전트를 위한 상세한 디자인 토큰 및 컴포넌트 스펙은 프로젝트 루트의 `DESIGN.md`를 참조합니다.

## 1. 디자인 철학

- "따뜻한 보라(Warm Purple)"를 주조색으로 사용하는, 친근하고 가벼운 도구 느낌의 디자인입니다.
- 핵심 키워드: 따뜻함, 가벼움, 깔끔함, 신뢰감, 저밀도
- 정보 밀도를 낮게 유지하고, 충분한 여백으로 시각적 여유를 줍니다.
- 색상은 최소한으로 사용하며, primary(보라) 외의 색상은 상태 피드백(성공/경고/오류) 용도로만 제한합니다.

## 2. UI 라이브러리

- 컴포넌트: `shadcn/ui`를 사용합니다. 커스텀 컴포넌트를 직접 만들기 전에 shadcn에서 제공하는 컴포넌트를 먼저 확인합니다.
- 아이콘: `lucide-react`를 사용합니다. 새로운 아이콘이 필요할 경우 lucide에서 먼저 찾습니다.
- shadcn 컴포넌트의 스타일을 커스터마이징할 때는 `globals.css`의 CSS 변수를 수정하는 방식을 우선합니다. 컴포넌트 소스를 직접 수정하는 것은 마지막 수단입니다.

## 3. 색상 체계

shadcn/ui의 CSS 변수 체계를 기반으로 하되, "따뜻한 보라" 팔레트로 커스터마이징합니다.

### 원칙

- 직접적인 색상 코드(HEX, RGB 등) 사용을 지양하고, `src/app/globals.css`에 정의된 CSS 시맨틱 변수를 최우선으로 사용합니다.
- shadcn의 CSS 변수 네이밍(`--background`, `--foreground`, `--primary`, `--card` 등)을 따릅니다.

### Tailwind 팔레트 우선 원칙

색상, 간격(Spacing) 등의 디자인 값을 새로 정할 때는 Tailwind CSS에서 제공하는 값을 우선적으로 탐색하여 적합한 값이 있으면 그것을 사용합니다. Tailwind 팔레트에 적절한 값이 없을 때에 한해 별도의 커스텀 값을 지정합니다.

예시:

```css
--link: #2563eb; /* blue-600 */
```

위와 같이 Tailwind 팔레트에서 가져온 값이라면, HEX 코드 옆에 주석으로 Tailwind 색상명을 명시합니다.

### 주요 색상 방향

- Primary: 따뜻한 보라 (`#4A3F6B` 계열, 드래프트 참고)
- Accent: 소프트 핑크 (`#D4708A` 계열, 포인트 강조용)
- Background: 연한 라벤더 톤의 회색-흰색 (`#F7F5FB` 계열)
- Surface: 순백 (`#FFFFFF`)
- 상태 색상: 성공(녹색), 경고(황색), 오류(적색)는 별도 정의

## 4. 타이포그래피

- 기본 글꼴: `Pretendard` 웹 폰트를 전역으로 사용합니다 (`font-family: 'Pretendard', sans-serif`).
- 단어 줄바꿈 시 레이아웃이 깨지는 것을 방지하기 위해 `word-wrap: break-word` 속성이 기본 적용되어 있습니다.
- 타이포그래피 계층 구조(Heading, Body, Caption 등)의 구체적인 스펙은 `DESIGN.md`를 참조합니다.

## 5. 반응형 웹 규칙

### 최소 지원 너비 (Minimum Width)

- 최소 지원 너비: `320px`
- 모든 페이지의 루트 요소 또는 래퍼(Wrapper) 요소는 최소 너비를 `320px`로 설정해야 합니다 (`min-width: 320px`).
- 스크롤 정책: 디바이스 또는 브라우저 창의 너비가 320px 미만으로 좁아질 경우, 억지로 UI를 구겨 넣어서 레이아웃이 깨지게 두지 않고, 320px의 레이아웃을 유지하면서 자연스럽게 좌우 스크롤이 발생하도록 처리합니다.

### 용어 정의 (Terminology)

디자인 가이드에서 사용하는 화면 방향(Orientation)에 대한 용어 정의입니다.

- "Portrait": 기기를 세로로 길게 세워 든 상태 (세로 모드)
- "Landscape": 기기를 가로로 넓게 눕혀 든 상태 (가로 모드)

### 브레이크포인트 (Breakpoints)

디바이스의 다양성을 고려하여 뷰포트를 4단계로 구분하고, Tailwind CSS의 브레이크포인트 스케일을 활용하여 반응형 레이아웃을 설계합니다.

#### 모바일 (Mobile)

- `sm` 미만 (`< 640px`)
- 1단 세로형 배치를 기본으로 하며, 화면을 최대한 활용하기 위해 좌우 여백을 최소화합니다.
- 일반적인 스마트폰의 "Portrait" 방향에 해당합니다.

#### 태블릿 (Tablet)

- `sm` 이상 `md` 미만 (`640px ~ 767px`)
- `sm:` 접두사를 활용하여 모바일보다 약간 더 여유 있는 배치를 시작합니다.
- 대화면 스마트폰(iPhone Pro Max, Galaxy S Ultra 등)의 Landscape 혹은 소형 태블릿(iPad Mini 등)의 Portrait 방향에 해당합니다.

#### 소형 데스크탑 (Small Desktop)

- `md` 이상 `lg` 미만 (`768px ~ 1023px`)
- 본격적인 가로 방향 배치(e.g., `md:flex-row`)를 적용하기 시작하는 구간입니다.
- 일반적인 태블릿(아이패드 등)의 "Portrait" 방향에 해당합니다.

#### 대형 데스크탑 (Large Desktop)

- `lg` 이상 (`1024px ~`)
- 화면이 과도하게 넓어지는 것을 방지하기 위해 컨테이너의 최대 너비를 제한하고 중앙 정렬(e.g., `lg:max-w-body lg:mx-auto`)을 적용합니다.
- 일반적인 태블릿의 "Landscape" 및 노트북, 모니터 화면에 해당합니다.

## 6. Do's and Don'ts

### Do

- shadcn의 CSS 변수를 통해 색상을 지정할 것
- 컴포넌트 간 간격에 Tailwind spacing 유틸리티를 사용할 것
- 모든 인터랙티브 요소에 focus 상태를 유지할 것 (shadcn 기본 설정 유지)
- 상태 피드백(로딩, 성공, 오류)을 사용자에게 명확히 전달할 것

### Don't

- CSS에 HEX/RGB 색상을 직접 하드코딩하지 말 것
- shadcn 컴포넌트가 있는데 같은 역할의 커스텀 컴포넌트를 만들지 말 것
- lucide-react 외의 아이콘 라이브러리를 혼용하지 말 것
- 같은 뷰에 3가지 이상의 강조 색상을 동시에 사용하지 말 것
