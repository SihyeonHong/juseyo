# 디자인 가이드

프로젝트 전반의 일관성 있는 UI/UX를 유지하기 위한 퍼블리싱 및 디자인 표준 규칙을 정의합니다.

## 1. 반응형 웹 규칙

### 최소 지원 너비 (Minimum Width)

- 최소 지원 너비: `320px`
- 모든 페이지의 루트 요소 또는 래퍼(Wrapper) 요소는 최소 너비를 `320px`로 설정해야 합니다 (`min-width: 320px`).
- 스크롤 정책: 디바이스 또는 브라우저 창의 너비가 320px 미만으로 좁아질 경우, 억지로 UI를 구겨 넣어서 레이아웃이 깨지게 두지 않고, 320px의 레이아웃을 유지하면서 자연스럽게 '좌우 스크롤(Horizontal Scroll)'이 발생하도록 처리합니다.

### 용어 정의 (Terminology)

디자인 가이드에서 사용하는 화면 방향(Orientation)에 대한 용어 정의입니다.

- "Portrait": 기기를 세로로 길게 세워 든 상태 (세로 모드)
- "Landscape": 기기를 가로로 넓게 눕혀 든 상태 (가로 모드)

### 브레이크포인트 (Breakpoints)

디바이스의 다양성을 고려하여 뷰포트를 4단계로 구분하고, Tailwind CSS의 브레이크포인트 스케일을 활용하여 반응형 레이아웃을 설계합니다.

#### 모바일 (Mobile)

- `sm` 미만 (`< 640px`)
- 1단 세로형 배치를 기본으로 하며, 화면을 최대한 활용하기 위해 좌우 여백을 최소화합니다.
- 일반적인 스마트폰의 "Portrait" 방향에 해당합니다.

#### 태블릿 (Tablet)

- `sm` 이상 `md` 미만 (`640px ~ 767px`)
- `sm:` 접두사를 활용하여 모바일보다 약간 더 여유 있는 배치를 시작합니다.
- 대화면 스마트폰(iPhone Pro Max, Galaxy S Ultra 등)의 Landscape 혹은 소형 태블릿(iPad Mini 등)의 Portrait 방향에 해당합니다.

#### 소형 데스크탑 (Small Desktop)

- `md` 이상 `lg` 미만 (`768px ~ 1023px`)
- 본격적인 가로 방향 배치(e.g., `md:flex-row`)를 적용하기 시작하는 구간입니다.
- 일반적인 태블릿(아이패드 등)의 "Portrait" 방향에 해당합니다.

#### 대형 데스크탑 (Large Desktop)

- `lg` 이상 (`1024px ~`)
- 화면이 과도하게 넓어지는 것을 방지하기 위해 컨테이너의 최대 너비를 제한하고 중앙 정렬(e.g., `lg:max-w-body lg:mx-auto`)을 적용합니다.
- 일반적인 태블릿의 "Landscape" 및 노트북, 모니터 화면에 해당합니다.

## 2. 디자인 토큰 및 시맨틱 변수

직접적인 색상 코드(HEX, RGB 등) 사용을 지양하고, 다크 모드(Dark Mode) 및 테마 확장을 유연하게 지원하기 위해 `src/app/globals.css`에 정의된 CSS 시맨틱 변수를 최우선으로 사용합니다.

### Tailwind 팔레트 우선 원칙

색상, 간격(Spacing) 등의 디자인 값을 새로 정할 때는 Tailwind CSS에서 제공하는 값을 우선적으로 탐색하여 적합한 값이 있으면 그것을 사용합니다. Tailwind 팔레트에 적절한 값이 없을 때에 한해 별도의 커스텀 값을 지정합니다.

예시:

```css
--link: #2563eb; /* blue-600 */
```

위와 같이 Tailwind 팔레트에서 가져온 값이라면, HEX 코드 옆에 주석으로 Tailwind 색상명을 명시합니다.

### 주요 색상 토큰

- `var(--background)`: 애플리케이션의 기본 배경색 (라이트 모드: `#f8f8f6`, 다크 모드: `#09090b`)
- `var(--background-secondary)`: 카드, 모달 등 강조가 필요한 보조 배경색 (다크 모드에서 배경과 동일, 라이트 모드에서 `#ffffff`)
- `var(--foreground)`: 기본 텍스트 색상
- `var(--muted)`: 보조 텍스트 색상 (강조되지 않는 설명, 비활성 요소 등)
- `var(--bg-code)`: 코드 블록 또는 인라인 코드 배경색
- `var(--link)`: 하이퍼링크 및 클릭 가능한 텍스트 색상 (라이트 모드: `blue-600`, 다크 모드: `blue-400`)

## 3. 타이포그래피 (Typography)

- 기본 글꼴: `Pretendard` 웹 폰트를 전역으로 사용합니다 (`font-family: 'Pretendard', sans-serif`).
- 단어 줄바꿈 시 레이아웃이 깨지는 것을 방지하기 위해 `word-wrap: break-word` 속성이 기본 적용되어 있습니다.

## 4. 아이콘 (Icons)

이 프로젝트에서는 아이콘 라이브러리로 `lucide-react`를 사용하고 있습니다.
새로운 아이콘이 필요할 경우, 먼저 `lucide-react`에서 제공하는 아이콘 중 적절한 것을 찾아 우선적으로 사용해야 합니다.
