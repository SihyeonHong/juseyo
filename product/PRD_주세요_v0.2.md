## 1. 배경 및 목적

### 1.1 문제 정의

중고거래를 2개 이상의 플랫폼(당근, 번개장터, 중고나라 등)에서 동시에 운영하는 판매자는 다음과 같은 반복적인 비효율을 경험한다.

1. 플랫폼마다 동일한 상품을 개별 등록해야 하는 중복 노동
2. 판매 중인 상품 목록을 한 곳에서 통합 확인하기 어려움
3. 상품 이미지 편집 도구가 분산되어 있어 작업 흐름이 끊김
4. 구매자에게 전체 판매 목록을 한 번에 공유할 수 있는 수단이 없음

### 1.2 해결 방향

플랫폼에 종속되지 않는 중립적인 상품 관리 도구를 제공한다. 판매자가 상품을 1회 등록하면 이미지 편집, 공개 마켓 페이지를 통한 구매자 공유, 장바구니형 거래 보조까지 한 곳에서 처리할 수 있도록 한다. 편집된 이미지는 다운로드하여 각 플랫폼에 직접 업로드하거나, 공개 마켓 페이지 URL을 공유하는 방식으로 플랫폼 연동을 대체한다.

### 1.3 제품 한 줄 정의

> 중고 판매자를 위한 플랫폼 중립 올인원 상품 관리 도구

## 2. 목표 및 성공 지표

### 2.1 런칭 목표

- MVP 배포 즉시 가입자 확보 시작
- 초기 수익 모델: 구글 애드센스 — 애드센스 심사 통과에 충분한 트래픽 확보 이후 적용. 초기에는 하단 배너 등 사용자 경험을 최소한으로 해치는 방식부터 시작

### 2.2 핵심 지표 (Key Metrics)

| 지표                  | 설명                                     | 목표 기준 |
| --------------------- | ---------------------------------------- | --------- |
| 등록 전환율           | 가입 후 상품 1건 이상 등록한 사용자 비율 | 추후 설정 |
| WAU                   | 주간 활성 판매자 수                      | 추후 설정 |
| 프로필 페이지 방문 수 | 공유된 판매자 페이지 UV                  | 추후 설정 |
| 이미지 편집 사용률    | 전체 상품 등록 중 편집 기능 사용 비율    | 추후 설정 |
| 광고 CTR              | 애드센스 적용 이후 수익 지표로 추가 예정 | —         |

> 참고: MAU 1,000명 달성 시 유료 플랜 출시를 검토한다.

## 3. 사용자 페르소나

### 3.1 메인 타깃

- 2개 이상의 플랫폼에서 동시에 판매하는 활성 이용자
- 판매 상품 수가 많아 관리 부담을 느끼는 개인 판매자
- 상품 사진 품질에 신경 쓰는 판매자

### 3.2 얼리어답터

- 번개장터·당근·중고나라를 동시에 운영하는 멀티 플랫폼 판매자
- 의류·전자기기·굿즈 등 이미지가 판매에 직결되는 카테고리 판매자

## 4. 기능 요구사항

> 우선순위: P0 = MVP 필수 / P1 = 초기 배포 후 / P2 = 이후 단계

### 4.1 상품 관리

| 기능              | 설명                                                                                    | 우선순위 |
| ----------------- | --------------------------------------------------------------------------------------- | -------- |
| 상품 등록         | 제목, 가격, 설명, 이미지, 카테고리 입력                                                 | P0       |
| 상품 목록 조회    | 전체 상품 목록, 판매 상태별 필터 (판매중 / 판매완료 / 숨김)                             | P0       |
| 상품 수정 / 삭제  | 등록된 상품 편집 및 삭제                                                                | P0       |
| 상품 고유 ID 부여 | 상품마다 고유 ID 자동 부여 — 구매자·판매자 간 상품 식별 및 장바구니형 워크플로우에 활용 | P0       |

### 4.2 이미지 편집

| 기능                  | 설명                                                               | 우선순위 |
| --------------------- | ------------------------------------------------------------------ | -------- |
| 기본 이미지 편집      | 이미지 분할 및 합치기 지원 (Sharp 기반, NestJS 처리)               | P0       |
| 중고거래 특화 편집 UX | 워터마크, 텍스트 삽입 등 판매 이미지에 최적화된 편집               | P1       |
| 편집 이미지 다운로드  | 편집 완료 이미지를 로컬로 다운로드 — 각 플랫폼 직접 업로드 시 활용 | P0       |

### 4.3 판매자 공개 프로필

| 기능                 | 설명                                                         | 우선순위 |
| -------------------- | ------------------------------------------------------------ | -------- |
| 공개 마켓 페이지     | `/:userId` 경로로 접근 가능한 판매자별 공개 상품 목록 페이지 | P0       |
| 마켓 페이지 URL 공유 | 판매자가 자신의 마켓 페이지 링크를 구매자에게 공유           | P0       |

### 4.4 장바구니형 거래 보조

| 기능             | 설명                                            | 우선순위 |
| ---------------- | ----------------------------------------------- | -------- |
| 구매자 상품 선택 | 공개 페이지에서 상품 선택 → 합산 금액 자동 계산 | P0       |
| ID 목록 복사     | 선택 상품 ID 목록을 복사해 판매자에게 전달      | P0       |
| 판매자 일괄 처리 | ID 목록 붙여넣기로 상품 일괄 선택·상태 변경     | P0       |

### 4.5 사용자 인증

| 기능              | 설명                                      | 우선순위 |
| ----------------- | ----------------------------------------- | -------- |
| 회원가입 / 로그인 | 이메일 기반 인증                          | P0       |
| 소셜 로그인       | Google / Kakao 등 (제공자 범위 추후 확정) | P2       |

다음 항목은 현재 버전의 범위에 포함되지 않는다.
