# Lean Canvas

## 1. Problem

1. 멀티 플랫폼 판매자는 각 플랫폼 양식에 맞게 상품을 개별 등록해야 한다.
2. 판매 현황 및 가격 변동을 플랫폼마다 수동으로 반영해야 한다.
3. 구매자에게 전체 판매 목록을 한 번에 공유할 수단이 없어 배송비 절약형 묶음 거래가 어렵다.

## 2. Customer Segments

- 2개 이상의 플랫폼(당근, 번개장터, 포카마켓, 트위터 등)에서 동시에 판매하는 활성 개인 판매자
- 판매 상품 수가 많아 관리 부담을 느끼는 판매자

얼리어답터: 수십 장 이상의 포토카드를 개별 판매하는 멀티 플랫폼 판매자

## Unique Value Proposition

중고 판매자를 위한 플랫폼 중립 올인원 상품 관리 도구

1회 등록으로 이미지 편집, 구매자 공유, 묶음 거래 보조까지 한 곳에서 처리한다.

## Solution

1. 상품 1회 등록 후 편집된 이미지를 다운로드해 각 플랫폼에 직접 업로드
2. 판매자 공개 마켓 페이지 URL을 구매자에게 공유
3. 장바구니형 워크플로우로 묶음 거래 보조 (상품 ID 기반 선택 → 합산 → 일괄 상태 변경)

## Channels

- 중고거래 플랫폼(당근, 번개장터 등) 이용자 대상 유입
- SNS 공유 — 판매자가 자신의 마켓 페이지 링크를 직접 공유, 구매자 유입과 동시에 서비스 노출
- SEO — 공개 상품·마켓 페이지 검색 노출
- 앱스토어 / 구글 플레이 (모바일 앱 대응 시)

## Revenue Streams

- 초기: Google AdSense (트래픽 확보 후 적용, 하단 배너 등 UX 최소 침해 방식 우선)
- MAU 1,000 달성 시 유료 플랜(프리미엄 구독) 출시 검토

## Cost Structure

고정 비용

- 서버·인프라 비용 (Vercel, AWS 등)
- 도메인·SSL 유지비

변동 비용

- 이미지 처리 및 스토리지 비용 — Google Cloud Storage, 업로드량 비례
- 광고 집행비 (선택적)

## Key Metrics

| 지표 | 설명 |
|---|---|
| 등록 전환율 | 가입 후 상품 1건 이상 등록한 사용자 비율 |
| WAU | 주간 활성 판매자 수 |
| 프로필 페이지 방문 수 | 공유된 판매자 페이지 UV |
| 이미지 편집 사용률 | 전체 상품 등록 중 편집 기능 사용 비율 |
| 광고 CTR | 애드센스 적용 이후 수익 지표 |

## Unfair Advantage

- 이미지 편집에 특화된 UX — 중고거래 맥락에 최적화된 편집 플로우
- 플랫폼 중립적 구조 — 거래에 미개입함으로써 기존 플랫폼과 경쟁하지 않고 보완재로 포지셔닝
- 판매자·구매자 양방향 ID 기반 워크플로우 — 타 플랫폼에서 재현하기 어려운 연동 UX
- 1인 개발 → 사용자 피드백 반영 속도 우위

# Strategy Descriptions

린 캔버스는 이 프로젝트의 제1 source로 활용된다. 
이 섹션에서는 위의 린 캔버스를 보완하는 추가 내용을 다룬다. 

## 1. Problem

직접 결제나 채팅 통합 없이, 판매자의 관리 워크플로우만 중앙화하는 것이 핵심 접근이다. 플랫폼과 경쟁하지 않고 그 위에서 작동하는 도구를 지향한다.

## 2. Customer Segments

### 얼리어답터를 포카 판매자로 설정한 이유
#### 1) 뚜렷한 pain point
아이돌 포토 카드는 일반적으로 수십~수백 장의 규격 사이즈 상품을 한꺼번에 하나의 사진에 찍어서 거래한다. 그러나 일괄 구매보다는 개별 구매 비중이 높고, 각 상품별 시세 변동도 심하다. 

각 포카별 시세는 포카마켓에서 알 수 있으나, 타 플랫폼 대비 시세가 높은 편이라 구매자 선호도가 떨어지며 개별 거래만 가능해 배송비 부담이 크다는 단점이 있다. 그래서 주로 개별 거래 시 "판매하시는 나띠 포카 더 있으신가요?"등의 추가 질문이 이어지기도 한다 - 바로 이 부분이 번거로운 지점이다. 

가장 포카 및 아이돌 굿즈 거래가 활발한 플랫폼은 번개장터이다. 그러나 번개장터는 수수료가 발생하는 자체 안전결제 사용을 강제하면서도 막상 안전 보장 서비스가 미흡하여 많은 이용자들에게 비판받는다. 번개장터를 신뢰할 수 없어 타 연락 수단으로 이동하여 거래하기 위한 각종 우회 방법이 흔하게 공유되는 실정이다. 다른 플랫폼으로 이동하고 싶다는 불만의 목소리가 이미 높지만, 당근은 지역 기반이고 중고나라는 더 심각한 부실 운영, 이외에는 사용자가 적어 대체제가 없는 상황. 


타깃을 포카 판매자로 한정하지 않고 "멀티 플랫폼 중고 판매자 전반"으로 넓게 설정한 이유는, 포카 외 일반 중고거래, 특히 작은 물건 여러 가지를 개별로 한꺼번에 판매하는 경우에서도 동일한 pain point가 발생하기 때문이다.
#### 2) 높은 피드백 품질 및 고객 충성도 기대 
아이돌 포토카드 판매자는 대체로 다음과 같은 특성을 지닌다. 

- 10대 ~ 30대 여성 비중이 높다. 
- 트렌드에 민감하고, 커뮤니티 내 입소문 속도가 빠르다. 
- 구매력이 높다: 이미 아이돌 문화를 즐기기 위해 적지 않은 지출을 하고 있는 만큼 소프트웨어와 같은 무형의 서비스에 돈을 지불하는 것에 거부감이 없다. 
- 반복 이용 가능성이 높다.

## Unique Value Proposition

"플랫폼 중립"이 핵심 포지셔닝이다. 특정 플랫폼의 API 연동이나 제휴 없이도 작동하며, 판매자가 기존 플랫폼 사용 습관을 바꾸지 않아도 된다. 진입 장벽이 낮다.

## Solution

편집 이미지 다운로드 방식은 플랫폼 API 없이도 연동 효과를 낸다. 공개 마켓 페이지는 서비스 자체가 구매자에게 노출되는 채널이기도 하다. 장바구니형 워크플로우는 판매자-구매자 간 커뮤니케이션 비용을 줄이는 핵심 기능이다.

## Channels

현재 채널 전략은 미검증 가설이다. SNS 공유를 통한 판매자 마켓 페이지 노출이 구매자 유입과 서비스 노출을 동시에 달성하는 핵심 채널로 가정하고 있다. SEO는 공개 상품 페이지가 검색에 노출되는 구조를 전제로 하며, MVP 배포 이후 실제 유입 경로 데이터를 기반으로 채널 우선순위를 재조정할 예정이다. 모바일 앱 채널(앱스토어 등)은 현재 웹 우선 전략상 후순위다.

## Revenue Streams

AdSense를 선행하는 이유는 유료 플랜 전환에 필요한 MAU 임계치(1,000명)를 달성하기 전까지 수익 공백을 최소화하기 위해서다. 유료 플랜의 구체적인 기능 범위 및 가격 정책은 MAU 추이를 보며 결정한다.

## Cost Structure

고정 비용은 인프라(Vercel, AWS)와 도메인·SSL로 구성된다. 변동 비용의 핵심은 GCS 스토리지로, 업로드량에 비례하나 초기 트래픽 수준에서는 무시 가능한 수준으로 예상된다. 1인 개발 운영 구조상 인건비가 없어 초기 고정 비용 부담이 낮다.

## Key Metrics

목표 수치는 MVP 배포 후 초기 데이터를 보고 설정할 예정이다. 현재는 지표 정의만 확정된 상태다.

## Unfair Advantage

플랫폼 중립 구조는 기존 플랫폼과의 경쟁을 회피하는 동시에 보완재로 포지셔닝되는 핵심 우위다. ID 기반 양방향 워크플로우(구매자 선택 → 판매자 일괄 처리)는 단순 목록 공유 도구와 차별화되는 지점이며, 타 플랫폼에서 독립적으로 재현하기 어렵다. 이미지 편집 특화 UX는 중고거래 맥락에서 반복적으로 발생하는 작업(워터마크, 분할, 합치기)을 줄이는 실질적 우위다. 1인 개발 구조는 사용자 피드백을 빠르게 반영할 수 있는 운영 속도 우위로도 작용한다.
