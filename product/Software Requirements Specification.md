## 1. 개요

### 1.1 목적

이 문서는 "주세요" 시스템의 소프트웨어 요구사항을 정의한다. 개발, 테스트, 유지보수 단계에서 기능적/비기능적 요구사항의 기준 문서로 활용한다.

### 1.2 범위

"주세요"는 중고 판매자를 위한 플랫폼 중립 올인원 상품 관리 도구이다. 여러 플랫폼(당근, 번개장터, 포카마켓, 트위터 등)에서 동시에 판매하는 개인 판매자가 상품을 1회 등록한 뒤 이미지 편집, 구매자 공유, 묶음 거래 보조까지 한 곳에서 처리할 수 있도록 지원한다.

시스템은 다음 두 개의 독립적인 서비스로 구성된다.

- 프론트엔드 웹 애플리케이션 (Next.js)
- 백엔드 API 서버 (NestJS, 별도 레포지토리)

### 1.3 용어 정의

| 용어                  | 정의                                                                                             |
| --------------------- | ------------------------------------------------------------------------------------------------ |
| 판매자                | 시스템에 가입하여 상품을 등록하고 관리하는 사용자                                                |
| 구매자                | 판매자의 공개 마켓 페이지를 통해 상품을 열람하는 비인증 사용자                                   |
| 마켓 페이지           | 판매자별로 부여되는 공개 상품 목록 페이지 (`/:userId`)                                           |
| 장바구니형 워크플로우 | 구매자가 상품 선택 → ID 목록 복사 → 판매자에게 전달 → 판매자 일괄 처리하는 거래 보조 흐름        |
| 상품 ID               | 각 상품에 자동 부여되는 고유 식별자. 장바구니형 워크플로우에서 판매자-구매자 간 상품 식별에 사용 |

## 2. 시스템 개요

```
┌───────────────┐         ┌───────────────┐         ┌──────────────┐

       │                         │                        ▲
       │  공개 마켓 페이지 열람   │  상품 등록/관리         │ 이미지 다운로드 후

           ▼
```

## 3. 기능 요구사항

### 3.1 사용자 인증

#### FR-AUTH-001: 이메일 회원가입

- 우선순위: P0
- 설명: 사용자는 이메일과 비밀번호로 회원가입할 수 있다.
- 입력: 이메일 주소, 비밀번호
- 처리: 이메일 형식 검증, 비밀번호 규칙 검증, 중복 가입 확인
- 출력: 가입 성공 시 세션 생성 및 로그인 상태 전환

#### FR-AUTH-002: 이메일 로그인

- 우선순위: P0
- 설명: 가입된 이메일과 비밀번호로 로그인할 수 있다.
- 입력: 이메일 주소, 비밀번호
- 처리: 자격 증명 검증
- 출력: 인증 성공 시 세션 생성, 실패 시 오류 메시지

#### FR-AUTH-003: 로그아웃

- 우선순위: P0
- 설명: 로그인된 사용자는 로그아웃할 수 있다.
- 처리: 세션 무효화
- 출력: 비인증 상태로 전환, 홈 페이지 리다이렉트

#### FR-AUTH-004: 소셜 로그인

- 우선순위: P2
- 설명: Google, Kakao 등 외부 OAuth 제공자를 통해 로그인할 수 있다.
- 처리: OAuth 2.0 인증 흐름 수행
- 비고: 제공자 범위는 추후 확정

---

### 3.2 상품 관리

#### FR-PROD-001: 상품 등록

- 우선순위: P0
- 설명: 인증된 판매자는 새 상품을 등록할 수 있다.
- 입력: 제목, 가격, 설명, 이미지, 판매 상태, 태그
- 처리: 상품 고유 ID 자동 부여, 이미지 업로드 시 GCS 저장
- 출력: 등록된 상품 정보 반환
- 초기 상태: "판매중"

#### FR-PROD-002: 상품 목록 조회

- 우선순위: P0
- 설명: 판매자는 자신의 전체 상품 목록을 조회할 수 있다.
- 필터: 판매 상태별 (판매중 / 판매완료 / 비공개 / 거래중)
- 출력: 상품 목록 (제목, 가격, 상태, 썸네일, 상품 ID)

#### FR-PROD-003: 상품 수정

- 우선순위: P0
- 설명: 인증된 판매자는 자신의 상품 정보를 수정할 수 있다.
- 수정 가능 필드: 제목, 가격, 설명, 이미지, 판매 상태, 태그
- 제한: 본인 소유 상품만 수정 가능

#### FR-PROD-004: 상품 삭제

- 우선순위: P0
- 설명: 인증된 판매자는 자신의 상품을 삭제할 수 있다.
- 처리: 연관 이미지 GCS에서 삭제, 데이터베이스 레코드 삭제
- 제한: 본인 소유 상품만 삭제 가능

#### FR-PROD-005: 상품 고유 ID 부여

- 우선순위: P0
- 설명: 상품 등록 시 시스템이 고유 ID를 자동으로 부여한다.
- 용도: 구매자-판매자 간 장바구니형 워크플로우에서 상품 식별에 사용

---

### 3.3 이미지 편집

#### FR-IMG-001: 이미지 분할

- 우선순위: P0
- 설명: 업로드된 이미지를 지정 기준으로 분할할 수 있다.
- 처리: NestJS 서버에서 Sharp 라이브러리로 처리

#### FR-IMG-002: 이미지 합치기

- 우선순위: P0
- 설명: 여러 이미지를 하나로 합칠 수 있다.
- 처리: NestJS 서버에서 Sharp 라이브러리로 처리

#### FR-IMG-003: 편집 이미지 다운로드

- 우선순위: P0
- 설명: 편집이 완료된 이미지를 로컬 디바이스로 다운로드할 수 있다.
- 용도: 다운로드한 이미지를 각 판매 플랫폼에 수동 업로드

#### FR-IMG-004: 가격 표시

- 우선순위: P0
- 설명: 각 상품별 이미지에 가격 표시 텍스트 이미지를 추가 편집할 수 있다.
- 처리: NestJS 서버에서 Sharp 라이브러리로 처리

#### FR-IMG-005: 중고거래 특화 편집

- 우선순위: P1
- 설명: 워터마크 삽입, 텍스트 오버레이 등 중고거래에 최적화된 편집 기능을 제공한다.
- 비고: 상세 기능 범위는 추후 확정

---

### 3.4 판매자 공개 프로필

#### FR-MARKET-001: 공개 마켓 페이지

- 우선순위: P0
- 설명: 각 판매자는 `/:userId` 경로로 접근 가능한 공개 상품 목록 페이지를 보유한다.
- 표시 정보: 판매자 정보, 판매중인 상품 목록 (제목, 가격, 이미지, 상품 ID)
- 접근: 인증 불필요

#### FR-MARKET-002: 마켓 페이지 URL 공유

- 우선순위: P0
- 설명: 판매자는 자신의 마켓 페이지 URL을 복사하여 외부에 공유할 수 있다.
- 출력: 클립보드에 URL 복사, 공유 성공 피드백 (toast)

---

### 3.5 장바구니형 거래 보조

#### FR-CART-001: 구매자 상품 선택

- 우선순위: P0
- 설명: 구매자는 공개 마켓 페이지에서 원하는 상품을 선택할 수 있다.
- 처리: 선택 상품의 합산 금액 자동 계산
- 출력: 선택 상품 수, 합산 금액 표시

#### FR-CART-002: 상품 ID 목록 복사

- 우선순위: P0
- 설명: 구매자는 선택한 상품들의 ID 목록을 클립보드에 복사할 수 있다.
- 용도: 복사한 ID 목록을 외부 메신저 등을 통해 판매자에게 전달

#### FR-CART-003: 판매자 일괄 처리

- 우선순위: P0
- 설명: 판매자는 구매자로부터 전달받은 ID 목록을 붙여넣어 해당 상품을 일괄 선택하고 상태를 변경할 수 있다.
- 입력: 상품 ID 목록 (붙여넣기)
- 처리: ID 매칭 → 상품 일괄 선택 → 상태 변경 (예: 판매중 → 판매완료)

---

### 3.6 국제화

#### FR-I18N-001: 다국어 지원

- 우선순위: P1
- 설명: 시스템은 한국어(기본) 및 영어를 지원한다.

#### FR-I18N-002: 언어 전환

- 우선순위: P1
- 설명: 사용자는 UI를 통해 언어를 전환할 수 있다.
- 동작: locale 전환 시 URL이 변경되며 페이지가 해당 언어로 렌더링된다.

## 5. 외부 인터페이스 요구사항

### 5.1 사용자 인터페이스

- 디자인 시스템: [DESIGN.md](../DESIGN.md) 참조

### 5.2 하드웨어 인터페이스

해당 없음. 웹 기반 애플리케이션.

### 5.3 소프트웨어 인터페이스

| 시스템               | 역할                                    | 통신 방식       |
| -------------------- | --------------------------------------- | --------------- |

- 인증: 쿠키 기반 세션
- 오류 코드: HTTP 표준 상태 코드 (400, 401, 403, 404, 500)

## 6. 데이터 요구사항

### 6.1 논리적 데이터 모델

#### 사용자 (User)

| 필드       | 타입     | 필수   | 설명                               |
| ---------- | -------- | ------ | ---------------------------------- |
| id         | string   | 예     | 고유 식별자                        |
| email      | string   | 예     | 이메일 주소                        |
| name       | string   | 예     | 사용자 표시 이름                   |
| avatar     | string   | 아니오 | 프로필 이미지 URL                  |
| provider   | string   | 예     | 인증 제공자 (email, google, kakao) |
| created_at | datetime | 예     | 가입 일시                          |

#### 상품 (Product)

| 필드        | 타입     | 필수   | 설명                                      |
| ----------- | -------- | ------ | ----------------------------------------- |
| id          | string   | 예     | 상품 고유 ID (장바구니 워크플로우에 사용) |
| user_id     | string   | 예     | 소유 판매자 ID                            |
| title       | string   | 예     | 상품 제목 (미기입 시 '제목 없음')         |
| price       | number   | 예     | 상품 가격 (미기입 시 '가격 미정')         |
| description | string   | 아니오 | 상품 설명                                 |
| image_url   | string   | 아니오 | 상품 이미지 URL (GCS)                     |
| tags        | string[] | 아니오 | 상품 태그 목록                            |
| status      | enum     | 예     | 판매중 / 판매완료 / 비공개 / 거래중       |
| created_at  | datetime | 예     | 등록 일시                                 |
| updated_at  | datetime | 예     | 최종 수정 일시                            |

### 6.2 데이터 보존

- 상품 삭제 시 관련 이미지도 GCS에서 삭제한다.
- 사용자 탈퇴 시 처리 정책은 추후 정의한다.

## 7. 우선순위 및 릴리스 계획

### 7.1 MVP (P0) — 첫 번째 릴리스

- 이메일 기반 회원가입/로그인/로그아웃
- 상품 CRUD (등록, 조회, 수정, 삭제)
- 기본 이미지 편집 (분할, 합치기) 및 다운로드
- 판매자 공개 마켓 페이지
- 장바구니형 거래 보조 (구매자 선택 → ID 복사 → 판매자 일괄 처리)
- 라이트/다크 테마

### 7.2 P1 — 두 번째 릴리스

- 중고거래 특화 이미지 편집 UX (워터마크, 텍스트 삽입)
- 다국어 지원 (한국어, 영어)

### 7.3 P2 — 이후 릴리스

- Google AdSense 적용
- 소셜 로그인 (Google, Kakao)
- 판매 통계 대시보드
- 모바일 네이티브 앱 검토
